Firts day

The things I learned during my first time using html were a little bit intimidating at first, because of how the syntax order of the html works. The easy ones were header, h1, h2, h3, head, body, and p, the hard ones were ol, ul, li, and a. Then I learnt where they all went and that just placing them anywhere will make the site disorganized. Using github for the first time was easier than I thought, like learning how to create a repository, making it public, getting the link to the files inside its folder, and committing changes to it.

More things I learned was how to use embed css into html. Knowing how to add colors to the heading and paragraphs was easy, changing the color of the background was even easier. When it came to the div id and class that's when it became a little hard expecially when I had to use # or . when I was adding the changes of the div id and class to the css.

The assignment for lab05 was a little confusing at first. When I started to understand it creating the links that lead to different part of the page was the easy part. The hard part was getting the website to look good on my phone and my PC.

Little things that I learnt

  2. Using a consistent Header, main, and footer method inside the body.
